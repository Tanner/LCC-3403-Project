How Strong Induction Relates to Computer Science

The goal of CS 2050 is get CS majors to think logically, which is a critical step to becoming a good programmer. With strong induction, there's a clear parallel with good program design. Take for example the problem of writing a function that returns the list of coins that are needed for a stamp price. Someone who originally is given this problem might not be used to thinking about tackling issues like this using code and data reusability. He/she may write the following function:

def coinsForPrice(price):
if price is 12:
return [4, 4, 4]
if price is 13:
return [4, 4, 5]
if price is 14:
return [4, 5, 5]
if price is 15:
return [5, 5, 5]
if price is 16:
return [4, 4, 4, 4]
...

The problem here is that there's no way to handle all the possible cases, because you can't possibly write out each individual case. Instead, the programmer needs to realize there is an elegant solution that handles all of the cases, turning cases into a constant formula that will work for all prices greater than 15.

def coinsForPrice(price):
if price is 12:
return [4, 4, 4]
if price is 13:
return [4, 4, 5]
if price is 14:
return [4, 5, 5]
if price is 15:
return [5, 5, 5]
else if price >= 16:
return coinsForPrice(price - 4) + [4]

There are obvious parallels here between the parts of strong induction and the function above. Notice the basis step is essentially expressed with the first three conditions. The final condition is the recursive call that handles the rest of the cases we want to handle. Notice both of these are required in order for the function to work - the base cases are required.

It's important to say that this function is not an example of representing strong induction in code. This is really an example of recursion, but with strong induction, we can actually prove that this function will work for all values of stamp prices greater and equal to 12. That is the key point.
